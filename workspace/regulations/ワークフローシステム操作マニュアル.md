# ワークフローシステム操作マニュアル

発行日: 2025年10月1日
改定日: 2026年4月1日
作成: 情報システム部

---

## 1. 概要

本マニュアルは、購買業務ワークフローシステムの操作方法を説明するものです。本システムはファイルベースで運用されており、案件の管理はJSONファイルで行います。運用の詳細はワークフローシステム運用規程を参照してください。

---

## 2. 案件管理ファイルの作成

### 2.1 ファイルの保存場所

案件管理ファイルは以下に保存します。
- 案件ファイル: `/shared/workflow/cases/WF-YYYY-NNN.json`
- 関連文書: `/shared/workflow/docs/WF-YYYY-NNN_<文書種別>.json`

### 2.2 案件ID（採番ルール）

- 形式: `WF-YYYY-NNN`（例: WF-2026-001）
- YYYY: 西暦4桁
- NNN: 年度内の連番（001から開始）
- 採番時は、既存の案件ファイルを確認し、最大番号の次を使用してください

### 2.3 案件ファイルのフォーマット

```json
{
  "case_id": "WF-2026-001",
  "status": "draft",
  "request_id": "req_001",
  "requester": "製造部 田中",
  "item_summary": "品名の要約",
  "amount": 850000,
  "current_handler": "担当者名",
  "created_at": "2026-04-07T09:00:00",
  "updated_at": "2026-04-07T09:00:00",
  "history": [],
  "documents": {
    "purchase_request": null,
    "estimate": null,
    "approval_record": null,
    "order": null,
    "delivery_record": null,
    "inspection_record": null,
    "invoice": null,
    "payment_record": null
  },
  "approval_info": {
    "required_approver_level": "承認権限表に基づく",
    "actual_approver": null,
    "approval_date": null
  }
}
```

---

## 3. ステータスの確認方法

案件ファイルの `status` フィールドで現在の状態を確認できます。`current_handler` フィールドで、次に処理すべき担当者を確認できます。

### 3.1 ステータス一覧

| ステータス | 意味 | 次のアクション |
|-----------|------|-------------|
| draft | 下書き | 購買担当者が内容を確認し、承認依頼に進める |
| pending_approval | 承認待ち | 承認権限者が承認または差戻しを行う |
| approved | 承認済み | 購買担当者が発注書を作成する |
| rejected | 差戻し | 購買担当者が修正して再提出する |
| ordered | 発注済み | 取引先の納品を待つ |
| delivered | 納品済み | 依頼元が検収を行う |
| inspected | 検収済み | 経理担当者が請求書照合を行う |
| invoiced | 請求書受領 | 経理担当者が三者照合を行う |
| payment_processing | 支払処理中 | 経理担当者が支払を実行する |
| paid | 支払完了 | 案件完了 |

---

## 4. ステータスの更新方法

ステータスを更新する際は、以下の手順に従ってください。

1. 案件ファイルを読み込む
2. `current_handler` が自分であることを確認する
3. `status` を次のステータスに変更する
4. `current_handler` を次の担当者に変更する
5. `updated_at` を現在日時に更新する
6. `history` 配列に遷移記録を追加する（timestamp, action, actor, actor_role, from_status, to_status, comment）
7. 必要に応じて `documents` に関連ファイルのパスを記録する
8. ファイルを保存する

---

## 5. 承認処理の手順

1. `/shared/workflow/cases/` 内の案件ファイルを確認する
2. `status` が `pending_approval` の案件を探す
3. `current_handler` が自分であることを確認する
4. 案件の金額を確認し、自分の承認権限範囲内であることを確認する（承認権限表参照）
5. 承認の場合: `status` を `approved` に変更し、`approval_info` を更新する
6. 差戻しの場合: `status` を `rejected` に変更し、理由を `history` に記録する
7. `current_handler` を次の担当者に変更する

## 6. 文書ファイルの命名規則

| 文書種別 | ファイル名例 |
|---------|-----------|
| 購買依頼書 | WF-2026-001_request.json |
| 見積書 | WF-2026-001_estimate.json |
| 承認記録 | WF-2026-001_approval.json |
| 発注書 | WF-2026-001_order.json |
| 納品記録 | WF-2026-001_delivery.json |
| 検収記録 | WF-2026-001_inspection.json |
| 請求書 | WF-2026-001_invoice.json |
| 支払記録 | WF-2026-001_payment.json |

---

## 7. エラーへの対処

| 状況 | 対処方法 |
|------|---------|
| 案件ファイルが見つからない | システム管理者に連絡 |
| ステータスが不整合 | システム管理者に連絡。自分で直接修正しないこと |
| 必要な文書が紐付けられていない | 担当者に確認の上、文書を作成・紐付け |
| 案件IDが重複している | システム管理者に連絡 |

---

## 8. 注意事項

- **重要**: このシステムはファイルベースのため、操作規程を遵守してください。技術的にはファイルの直接編集が可能ですが、操作マニュアルに従わない操作は監査の対象となります。
- ステータスの遷移は必ず順序通りに行ってください。ステータスを飛ばして更新しないでください。
- `current_handler` に自分が設定されていない案件は処理しないでください。
- 案件の `history` は削除・改変しないでください。
- 不明点がある場合は、システム管理者または上長に確認してください。
